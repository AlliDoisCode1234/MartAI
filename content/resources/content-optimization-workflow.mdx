# Content Optimization Workflow: From Draft to Publish

**A step-by-step workflow for optimizing content before publishing.** Covers on-page SEO, readability, CTAs, and final quality checks.

## The Optimization Workflow

Content optimization transforms good writing into content that ranks and converts. Here's the complete process.

## Phase 1: Pre-Writing Optimization

### 1.1 Brief Review

Before writing, ensure:

- [ ] Primary keyword identified
- [ ] Search intent understood
- [ ] Competitor content analyzed
- [ ] Outline approved
- [ ] Word count target set

### 1.2 SERP Analysis

Study what's ranking:

| Rank | Title   | Word Count | Key Sections     |
| ---- | ------- | ---------- | ---------------- |
| #1   | [Title] | [count]    | [topics covered] |
| #2   | [Title] | [count]    | [topics covered] |
| #3   | [Title] | [count]    | [topics covered] |

Note gaps and opportunities.

## Phase 2: On-Page SEO

### 2.1 Title Tag Optimization

```html
<title>Primary Keyword - Secondary Keyword | Brand</title>
```

Guidelines:

- [ ] Under 60 characters
- [ ] Primary keyword near start
- [ ] Compelling and clickable
- [ ] Unique per page

### 2.2 Meta Description

```html
<meta
  name="description"
  content="Action-oriented description 
that includes primary keyword and compels clicks. 150-160 chars."
/>
```

Guidelines:

- [ ] 150-160 characters
- [ ] Includes primary keyword
- [ ] Has call-to-action
- [ ] Matches content accurately

### 2.3 Header Structure

```markdown
# H1: Main Topic (includes primary keyword)

## H2: Major Section 1

### H3: Subsection

## H2: Major Section 2

...
```

Guidelines:

- [ ] One H1 only
- [ ] Logical hierarchy
- [ ] Keywords in some H2s naturally
- [ ] H2s every 300-500 words

### 2.4 Keyword Placement

Natural keyword usage:

| Location        | Importance | Notes                       |
| --------------- | ---------- | --------------------------- |
| Title tag       | Critical   | Near beginning              |
| H1              | Critical   | Exact or close match        |
| First 100 words | High       | Natural inclusion           |
| H2 headers      | Medium     | Some variations             |
| Body text       | Medium     | 2-3 mentions per 1000 words |
| Image alt text  | Low        | Where relevant              |

> **Warning**: Never keyword stuff. Write for humans first.

### 2.5 URL Optimization

```
Good: /content-optimization-workflow
Bad: /post-12345?ref=draft
```

Guidelines:

- [ ] Short and descriptive
- [ ] Includes primary keyword
- [ ] Hyphen-separated
- [ ] No special characters

## Phase 3: Readability Optimization

### 3.1 Paragraph Length

- Maximum 3-4 sentences per paragraph
- Use single-sentence paragraphs for emphasis
- White space improves readability

### 3.2 Sentence Length

- Average under 20 words
- Mix short and medium sentences
- Break up complex ideas

### 3.3 Vocabulary

- Use simple words when possible
- Define technical terms
- Target grade 8-10 reading level

### 3.4 Formatting

Use visual variety:

- **Bold** for emphasis
- _Italics_ for terms
- `Code` for technical items
- > Blockquotes for callouts
- Lists for multiple points
- Tables for comparisons

## Phase 4: Content Completeness

### 4.1 Comprehensiveness Check

Compare against top-ranking content:

- [ ] All major subtopics covered
- [ ] Questions answered
- [ ] Objections addressed
- [ ] Examples provided
- [ ] Actionable steps included

### 4.2 Freshness Indicators

- [ ] Current year mentioned
- [ ] Recent statistics
- [ ] Updated best practices
- [ ] No outdated references

### 4.3 E-E-A-T Signals

- [ ] Author bio present
- [ ] Experience demonstrated
- [ ] Sources cited
- [ ] Expertise visible

## Phase 5: CTA Optimization

### 5.1 Primary CTA

Place your main call-to-action:

- After introduction (early engagement)
- Within content (contextual)
- At conclusion (after value delivery)

### 5.2 CTA Best Practices

| Do              | Don't               |
| --------------- | ------------------- |
| Be specific     | Vague "click here"  |
| Show value      | Focus on your needs |
| Create urgency  | Be pushy            |
| Test variations | Set and forget      |

## Phase 6: Technical Checks

### 6.1 Image Optimization

For each image:

- [ ] Descriptive filename (keyword-topic.jpg)
- [ ] Alt text with context
- [ ] Compressed file size
- [ ] Correct dimensions
- [ ] WebP or AVIF format

### 6.2 Internal Links

- [ ] 3-8 internal links per article
- [ ] Links to related content
- [ ] Varied anchor text
- [ ] Links from other content to this page

### 6.3 External Links

- [ ] Cite authoritative sources
- [ ] 2-4 external links minimum
- [ ] Open in new tab (optional)
- [ ] Check for broken links

## Phase 7: Final Quality Check

### Pre-Publish Checklist

**Content**

- [ ] Spelling and grammar checked
- [ ] All facts verified
- [ ] Reading flow is smooth
- [ ] Examples are relevant

**SEO**

- [ ] Title tag optimized
- [ ] Meta description written
- [ ] URL is clean
- [ ] Headers use keywords naturally

**Technical**

- [ ] Images optimized
- [ ] Links working
- [ ] Mobile preview checked
- [ ] Page speed acceptable

**Conversion**

- [ ] CTA is clear
- [ ] Value proposition strong
- [ ] Next steps obvious
